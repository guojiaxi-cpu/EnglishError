# 英语错题汇总

## 执行流程

1. 从项目 `scripts/paths.config.json` 读取 `vaultRoot`。
2. 运行以下只读采集命令，递归读取 `vaultRoot` 下全部 `.md` 文件：

```powershell
python .agents/skills/english-error-summary/scripts/collect_archived_errors.py --config scripts/paths.config.json
```

3. 检查脚本输出的 Markdown 文件数、题目记录数和读取警告。脚本失败时明确报告原因，不得凭空生成汇总。
4. 处理脚本输出的每一条 JSONL 记录，覆盖全部可读取笔记后再开始输出总结。
5. 按下述合并规则整理错误，直接在对话中输出最终结果。

## 归类与合并规则

- 按导致错误的最具体知识点或解题缺陷归类，例如：系动词后误用副词、比较级比较范围错误、固定搭配混淆、时态判断错误、词性转换错误、发音规则混淆、阅读定位错误。
- 只有底层规则、错误原因和纠正方法实质相同的记录才合并；不能仅因同属“语法”“词汇”或同一题型就合并。
- 同一道原题在多篇笔记中重复出现时只计算一次。使用“题目原文 + 学生错误答案 + 正确答案”判断重复。
- 同一错误类型下统计合并后的错误次数，并从原记录中选择最多 10 个有代表性的错误例句。
- 优先保留信息完整、能清楚体现错误原因的例句；例句必须来自归档原文，不得杜撰或改写成新的错误。
- 笔记格式不标准但仍能判断错误时纳入归类；无法确认错误内容时列入“待人工确认”，不要推测。

## 讲解要求

每个合并后的错误类型都必须包含：

- 明确、具体的错误类型名称。
- 错误的核心原因和容易混淆之处。
- 正确规则、结构或判断方法。
- 至少一个正确用法说明；优先使用原错误例句进行对照。
- 简洁的解题步骤或记忆方法。
- 最多 10 个原错误例句，并尽量列出原题、学生答案和正确答案。

## 输出格式

```markdown
# 英语错题汇总

**统计范围**：[vaultRoot]
**读取笔记**：X 篇
**归类后错误类型**：X 类

## 1. [具体错误类型]

**出现次数**：X 次

### 错误讲解
[核心原因、正确规则、判断方法和记忆方法]

### 原错误例句（最多 10 个）

1. **原题**：...
   **学生答案**：...
   **正确答案**：...

## 待人工确认
[仅在存在无法可靠分类的记录时输出]
```

错误类型按出现次数从高到低排列；次数相同时按错误类型名称排序。

## 只读约束

- 只读取 `vaultRoot` 下现有 Markdown，不修改、移动、重命名或删除任何文件。
- 不保存来源文件，不创建 `source` 或 `tmp` 内容。
- 不调用 `scripts/english_error_archive.py store-source` 或 `export-note`。
- 不创建汇总 Markdown，不写入 Obsidian；最终结果只在当前对话中直接输出。
